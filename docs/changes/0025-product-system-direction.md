# 同一前端陶瓷情报系统方向记录

## 背景

用户确认长期目标不是单一脚本，而是一个在同一前端中使用的陶瓷情报系统。当前项目只是其中一个核心板块：负责社媒趋势情报、报告生成、评分质检和后续多源接入。

## 记录内容

- 陶瓷机会雷达的时间窗口从 30/60/90 天扩展为 30/60/90/120 天。
- MediaCrawler 方向用于中国国内平台采集。个人自用阶段用户倾向直接参考/复用其模式并放入同一前端壳子；未来对外开放时需要重新评估许可证、合规、平台规则和替代实现。
- 共同数据库与数据分类总结工具后续可以接入大模型 API，用于主管式筛选、分类、审核、去噪、可信度判断。
- 最终输出不只限中文，还需要支持英文等其他语言，因此报告层未来应预留 `language` / `locale` 配置。

## 可行性判断

整体可行，但建议分阶段推进：

- 先稳定当前社媒趋势情报主线。
- 再做 YouTube tiny probe 和更多数据源。
- 再设计共同数据库。
- 最后把 MediaCrawler、陶瓷机会雷达和多语言输出纳入同一前端。

大模型 API 做筛选分类审核是可行的，但不能直接代替来源记录、证据分层和人工复核。

## 安全边界

- 本次只记录产品方向，不接入 MediaCrawler。
- 不新增爬虫。
- 不调用外部 API。
- 不修改正式报告逻辑。
- 不修改 `last30days-skill`。

## 后续

V0.8.0 仍建议优先做 YouTube tiny probe。陶瓷机会雷达可以作为 V0.9 或独立小版本启动，第一步可先建 `config/ceramic_events_sources.json` 和一个离线样例。
